Compliance & Standards

The Secure Intelligence Document Vault is engineered from the ground up to meet or exceed the most demanding government and enterprise security compliance frameworks. This page provides an overview of each standard and how the system addresses its requirements.

NIST SP 800-53 Rev. 5

Security and Privacy Controls for Federal Information Systems — the most comprehensive catalog of security controls for federal information systems and organizations. It covers 20 control families including Access Control (AC), Audit & Accountability (AU), Cryptographic Protection (SC), and Identification & Authentication (IA).

ℹ️ Applicability

Mandatory for all federal information systems and government contractors processing, storing, or transmitting sensitive federal data.

Key controls implemented:

  • AC-2: Account Management — automated provisioning and deprovisioning
  • AC-6: Least Privilege — RBAC with granular permission model
  • AU-2: Audit Events — comprehensive event logging
  • AU-10: Non-Repudiation — cryptographic proof of operations
  • IA-2: Identification and Authentication — MFA mandatory
  • SC-12: Cryptographic Key Establishment — HSM-backed key management
  • SC-13: Cryptographic Protection — FIPS-validated algorithms
  • SC-28: Protection of Information at Rest — AES-256-GCM

NIST SP 800-171 Rev. 3

Protecting Controlled Unclassified Information (CUI) — defines 110+ security requirements derived from NIST 800-53, organized into 14 requirement families. Required for all contractors and organizations handling CUI.

FIPS 140-3

Federal Information Processing Standard for Cryptographic Modules — validates that cryptographic implementations meet federal security requirements. The system targets Level 3 compliance, which adds tamper-resistance and identity-based authentication.

  • Level 1: Basic security — software cryptographic module ✓
  • Level 2: Tamper-evidence + role-based authentication ✓
  • Level 3: Tamper-resistance + identity-based authentication (target)
  • Uses Go's BoringCrypto fork for FIPS-validated cryptographic primitives
  • All algorithms are FIPS-approved: AES, SHA-256, HMAC, RSA

SOC 2 Type II

Service Organization Controls providing assurance over security, availability, processing integrity, confidentiality, and privacy. The system's comprehensive logging, access controls, and encryption directly support SOC 2 trust service criteria.

ISO 27001

International standard for Information Security Management Systems (ISMS). The system's architecture and operational controls are aligned to Annex A control objectives for access management, cryptography, operations security, and compliance.

Zero Trust Architecture (NIST SP 800-207)

Implementation follows the three core tenets:

  1. Verify explicitly — every request authenticated and authorized regardless of origin
  2. Use least privilege — just-in-time, just-enough access with RBAC + ABAC
  3. Assume breach — micro-segmentation, E2E encryption, continuous verification

Encryption & Cryptography

The Secure Intelligence Document Vault employs a defense-in-depth cryptographic strategy to ensure the confidentiality and integrity of all stored data. We strictly adhere to FIPS 140-3 standards for all cryptographic operations.

Data at Rest Encryption

All sensitive data stored on disk is encrypted using AES-256-GCM (Galois/Counter Mode). This provides both confidentiality (data is unreadable without the key) and integrity (tampering is detected).

✅ FIPS Validated

We use the FIPS-validated BoringCrypto module within Go for all cryptographic primitives.

Envelope Encryption Strategy

To support massive scale and secure key management, we use envelope encryption:

  1. Data Encryption Key (DEK): A unique 256-bit AES key is generated for every document version. This key encrypts the actual file content.
  2. Key Encryption Key (KEK): A master key stored in a Hardware Security Module (HSM) encrypts the DEK.
  3. Storage: The encrypted content and the encrypted DEK are stored together. The plaintext DEK is never stored on disk.

Data in Transit Encryption

All network communication is secured using TLS 1.3.

  • Protocol: TLS 1.3 (only)
  • Cipher Suites: Restricted to PFS (Perfect Forward Secrecy) suites:
    • TLS_AES_128_GCM_SHA256
    • TLS_AES_256_GCM_SHA384
    • TLS_CHACHA20_POLY1305_SHA256
  • Certificates: RSA-4096 or ECDSA P-384 signed by internal CA.
  • mTLS: Required for all service-to-service communication.

Role-Based Access Control (RBAC) Hierarchy

The system enforces a strict 7-level hierarchical RBAC model. Permissions serve as a baseline for access; higher-level roles inherit permissions from lower-level roles where appropriate, but specialized roles (like Auditor) are intentionally segregated.

Diagram

Access Control Model - RBAC Hierarchy

Role Definitions

Role Level Name Description
Level 1 SuperAdmin Full system control, critical config, emergency override.
Level 2 Admin User management, system configuration, routine ops.
Level 3 Manager Departmental oversight, approval workflows, delegation.
Level 4 User Standard read/write access to assigned documents.
Level 5 ReadOnly View-only access to assigned documents.
Level 6 Auditor Read-only access to audit logs and compliance reports only.
Level 7 SystemAccount Service-to-service authentication (automated).

Inheritance Model

ℹ️ Inheritance Rules

Roles inherit permissions from the level below them in the operational chain (User → Admin). The Auditor role is outside this chain to ensure separation of duties — Auditors cannot modify system data, and Admins cannot modify audit logs.

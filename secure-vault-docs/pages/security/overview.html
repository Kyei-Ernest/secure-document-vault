Security Overview

The security architecture of the Secure Intelligence Document Vault is built on Zero Trust principles, assuming that no user or network segment is inherently trusted. Security controls are layered (Defense in Depth) to ensure that the failure of any single control does not result in a system compromise.

STRIDE Threat Model

The system's security design is informed by a comprehensive STRIDE threat modeling exercise that identified and mitigated 113 distinct threats.

Cryptographic Standards

We adhere strictly to FIPS 140-3 requirements. No custom cryptography is used. All cryptographic operations utilize FIPS-validated modules.

  • Data at Rest: AES-256-GCM (Authenticated Encryption)
  • Data in Transit: TLS 1.3 only (Sweet32/POODLE mitigation)
  • Key Exchange: ECDHE (P-256, P-384 curves)
  • Hashing: SHA-256, SHA-384, Argon2id (for passwords)

Key Management Lifecycle

Secure key management is the foundation of our cryptographic security. We employ automated key rotation, strict separation of duties, and HSM-backed protection for all master keys.

Key Hierarchy

  1. Master Key (MK): The root of trust, stored exclusively in a FIPS 140-2 Level 3 Hardware Security Module (HSM). Never leaves the HSM boundary.
  2. Key Encryption Key (KEK): Used to encrypt Data Encryption Keys. Rotated every 90 days.
  3. Data Encryption Key (DEK): Unique per document version. Encrypts the actual data blob.

Automated Key Rotation

The system enforces automated rotation policies to limit the potential impact of a compromised key.

  • Master Keys: Rotated annually (re-encrypted KEKs).
  • KEKs: Rotated every 90 days (re-encrypted DEKs).
  • DEKs: Re-generated whenever a document is updated (new version).
ℹ️ Zero Downtime Rotation

Key rotation is performed online without service interruption. Old keys are retained for decryption of historical data but are deprecated for new encryption operations.

Secure Storage

Keys are never stored in plaintext on disk.

  • HSM: AWS KMS / Azure Key Vault / Google Cloud KMS for cloud deployments.
  • On-Premise: Thales Luna / YubiHSM support for on-prem deployments.
  • Memory Protection: Keys in memory are wiped (zeroed) immediately after use to prevent leakage via core dumps or swap.

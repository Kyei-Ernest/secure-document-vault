Technology Stack

The technology stack is chosen for security, performance, and auditability. We prioritize memory-safe languages, battle-tested cryptographic libraries, and ACID-compliant data stores.

Diagram

Technology Stack - Detailed Component Overview

Backend & Security

Component Technology Rationale
Language Go 1.21+ Memory safety, strong static typing, excellent concurrency, native compilation.
Crypto BoringCrypto / Go Standard Lib FIPS-validated cryptographic primitives, constant-time implementations.
Web Server Go net/http Production-ready standard library, no heavy framework overhead.

Data & Storage

  • PostgreSQL 15+: Object-Relational Database with robust Row-Level Security (RLS) and JSONB support.
  • Redis 7+: In-memory data structure store for high-performance caching and rate limiting.
  • S3-Compatible Object Storage: Scalable, durable storage for encrypted document files.

Frontend

  • React: Component-based UI library for dynamic interfaces.
  • TypeScript: Type-safety for frontend code, reducing runtime errors.
  • Tailwind CSS: Utility-first CSS framework for consistent, secure UI styling.

C4 Context Diagram (Level 1)

The Context Diagram shows the highest level of abstraction — the Secure Document Vault system as a single box, surrounded by its users and the external systems it interacts with. This diagram answers the question: "What does the system do and who uses it?"

Diagram

C4 Model — Context Diagram (Level 1)

Key Elements

  • End Users — government personnel who create, read, share, and manage classified documents
  • System Administrators — IT staff responsible for system configuration, user management, and monitoring
  • Auditors — compliance personnel who review audit logs and generate compliance reports
  • Key Management Service (KMS) — external HSM or cloud KMS for master key storage
  • MFA Provider — TOTP/FIDO2 authentication service
  • Email/SMS Gateway — notification delivery service

Important Notes

⚠️ Trust Boundaries

All external actors communicate with the system through the API Gateway, which enforces TLS 1.3, rate limiting, and request validation before any request reaches internal services.

  • All external communication is encrypted with TLS 1.3
  • KMS communication uses mTLS with certificate pinning
  • No direct access to internal services from external actors
